# 보안 정책 / Security Policy

## 지원 버전

| 버전 | 보안 패치 |
| ---- | --------- |
| 0.x  | ✅        |

`0.x` 는 활발히 변경되는 알파/베타 단계입니다. 최신 마이너 버전으로 업데이트하는 것을 권장합니다.

## 취약점 신고

**공개 이슈로 등록하지 마세요.** 보안 취약점은 비공개 채널로 보고해 주세요:

- 이메일: **security@terracelab.co.kr**
- (선호) GitHub Security Advisory: <https://github.com/terracelab/lore-ai/security/advisories/new>

다음을 포함해 주세요:

1. 취약점의 영향 범위 (예: RCE / 정보 노출 / DoS)
2. 영향받는 패키지·버전
3. 재현 단계 또는 PoC
4. (선택) 제안하는 패치

## 응답 SLA

- **48 시간 내**: 접수 확인 메일
- **7 일 내**: 영향 범위 평가 + 수정 일정 공유
- **30 일 내**: 패치 릴리즈 (심각도에 따라 단축)
- 공개: 패치 배포 후 GitHub Security Advisory 로 공시 + 신고자 크레딧 (원하는 경우)

## 책임 있는 공개

수정 패치가 npm 에 배포되기 전에 취약점을 공개하지 말아주세요. 책임 있는 공개에 협력해 주신 보안 연구자에게는 advisory 에 이름·핸들을 명시합니다 (옵트인).

## 의존성

런타임 의존성은 `npm audit` + Dependabot 으로 모니터링됩니다. 상위 패키지의 취약점이 발견되면 24~72 시간 내 패치 버전을 릴리즈합니다.
