# `lore synthesize [category]`

LLM 으로 L2 flow 파일을 재구성하기 위한 **프롬프트를 빌드** 합니다. 카테고리 인자를 생략하면 모든 카테고리를 한 프롬프트에 묶어 출력합니다.

```bash
lore synthesize                          # 모든 카테고리를 한 번에 — 가장 흔한 사용법
lore synthesize signal                   # 단일 카테고리만
lore synthesize --since 2.weeks          # 최근 2주 git log 포함
lore synthesize signal --since 1.month   # 단일 카테고리 + 한 달치 변경
lore synthesize --apply                  # (v0.2 예정) Anthropic API 호출 + 파일 덮어쓰기
```

## 출력물 — 강한 8섹션 구조

프롬프트는 LLM 이 다음 골격을 **그대로 따르도록** 강제합니다:

1. `## 1. 구성 앱/파일` — 서버/클라이언트 파일 풀리스트 (한 줄 책임)
2. `## 2. 데이터 모델` — ASCII ER 다이어그램 + 핵심 필드 제약 표
3. `## 3. 엔드포인트` — `Method | Path | View | 권한 | 핵심 로직` 5컬럼 표
4. `## 4. 대표 플로우` — `4.1`, `4.2` … 번호 매긴 순차 단계 (각 단계 = 한 액션)
5. `## 5. 권한·데코레이터 패턴`
6. `## 6. 클라이언트 상태` — store / context / hook / route guard
7. `## 7. 정책` — **확정 / ⚠️ 확인 필요 / TBD** 세 분류
8. `## 8. 관련 파일 인덱스` — 라인번호 포함 풀리스트

평면 산문 H2 는 금지하고 표·다이어그램·번호 단계로 재구조화하도록 명시. ⚠️ 마커는 정책 검토 필요·보안/금융 리스크 후보에만 사용.

## Connection 추론

어노테이션 태그가 없어도 LLM 이 다음 연결을 코드 단서로 추론해 4번 또는 8번 섹션에 명시합니다:

- 엔드포인트 ↔ 화면/페이지/스토어 액션 매핑
- 모델 ↔ 다른 모델의 FK / 참조 관계
- 서버 background task ↔ 트리거 모델·이벤트
- React 컴포넌트 ↔ 사용 hook ↔ API 클라이언트 함수

## YAML frontmatter 보존

LLM 출력은 **반드시 frontmatter (`---` 블록) 를 포함**하도록 강제합니다. Claude Code 처럼 파일을 직접 쓰는 에이전트 환경에서는 기존 파일 상단의 frontmatter 를 verbatim 보존하고 닫는 `---` 아래 본문만 새로 작성합니다. 신규 파일이거나 frontmatter 가 없으면 `slug` · `title` · `icon` · `order` · `summary` · `tags` · `last_reviewed` 를 직접 채웁니다. frontmatter 가 빠진 출력은 잘못된 결과물로 간주합니다 — `lore sync` 가 prepend 해주리라 기대하지 않습니다.

## `--since` git log 번들링

`--since <range>` 를 주면 카테고리 어노테이션이 위치한 디렉토리의 git log 가 프롬프트에 함께 들어갑니다. 파일별로 git 루트를 자동 탐지하므로 **모노레포 / 서브레포 / 멀티레포** 어느 구조든 동작합니다.

`<range>` 는 `git log --since` 가 받는 모든 형식: `2.weeks`, `1.month`, `2025-03-01`, `yesterday` 등.

```bash
lore synthesize --since 2.weeks    # 최근 2주
lore synthesize --since 1.month    # 한 달
lore synthesize --since 2026-01-01 # 절대 날짜
```

## 동작

### 단일 카테고리 모드

1. `domains[<category>]` 검증
2. 모든 프로젝트에서 해당 카테고리 어노테이션 추출
3. 기존 `.lore/flows/<category>.md` 본문 (있으면) 로드
4. (`--since` 시) 해당 카테고리 파일들의 git log 수집
5. L1 + 기존 L2 + L3 + git log → 8섹션 출력 contract 가 박힌 프롬프트 생성

### 전체 모드 (인자 없음)

1. `domains:` 의 모든 카테고리 순회
2. 각 카테고리의 어노테이션 + 기존 본문 수집 (어노테이션 0 인 카테고리는 skip)
3. (`--since` 시) 모든 카테고리의 파일에 대한 git log 한 번에 수집
4. 카테고리별 컨텍스트를 **하나의 프롬프트**로 묶고 `=== FILE: .lore/flows/<slug>.md ===` 마커로 LLM 이 응답을 분리하도록 지시

## 권장 워크플로 (v0)

`--apply` 가 v0.2 까지 미구현이므로:

```bash
lore synthesize --since 2.weeks | pbcopy   # macOS — 모든 카테고리 + git log 클립보드로
# Claude Code 에 붙여넣기 →
#   Claude Code 가 .lore/flows/<cat>.md 파일들을 직접 8섹션 구조로 갱신
```

이후 `lore sync` 를 한 번 더 실행해 frontmatter 를 갱신합니다.
