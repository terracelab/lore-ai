# L2 Frontmatter Contract

Lore Board 의 [`FlowMeta`](https://github.com/terracelab/lore-board/blob/main/src/lib/content.ts) 와 1:1 매핑됩니다. 추가 필드는 파서가 무시합니다.

```yaml
---
slug: signal # 필수 · 파일명과 동일
title: 시그널 # 필수 · 한글 표시명
icon: 📘 # 필수 · 이모지 1개
order: 2 # 필수 · 사이드바 정렬
summary: 시그널 톡·픽·리포트·투표
tags: [talk, realtime, pick]
last_reviewed: 2026-04-25
source_commit: abc1234
source_files: 23
---
```

## 본문 규칙

- `# H1` 1회 → `## H2` 다수
- H2 단위로 RAG 인덱싱됨
- H2 본문 ≥ 20 자 (너무 짧으면 인덱싱 skip)
- H3 이하는 본문 검색에만 포함

## INDEX.md

`lore sync` 가 `.lore/flows/INDEX.md` 로 자동 생성합니다. `slug: index` 로 마킹되어 Lore Board 의 `getAllFlows()` 가 사이드바에 노출하지 않습니다. 본문에는 다음이 포함됩니다:

- 카테고리 표 (#, slug, 아이콘, 이름, 문서 링크)
- frontmatter 스펙 (대시보드 파서용)
- L1 / L2 / L3 레이어 개념
- `lore sync` → `lore synthesize` → `lore review` 갱신 흐름

## 어디서 만들어지나

- L3 raw facts 는 [`lore sync`](/cli/sync) 가 `.lore/draft/<slug>.md` 에 적고 `last_reviewed` / `source_files` 를 자동 주입합니다.
- L2 보고서는 [`lore synthesize`](/cli/synthesize) 가 `.lore/flows/<slug>.md` 에 적습니다 (frontmatter 는 보존, 본문만 9섹션 골격으로 재작성).
- [`lore review`](/cli/review) 는 frontmatter 를 절대 건드리지 않고 본문만 부분 보강합니다.
