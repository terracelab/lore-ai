# Claude Code 통합

[Claude Code](https://claude.com/claude-code) 에서 `@lore` 로 도메인 컨텍스트를 자동 주입합니다.

## 설정

워크스페이스 루트에 `.mcp.json` 추가:

```json
{
  "mcpServers": {
    "lore": {
      "command": "npx",
      "args": ["-y", "lore-ai", "mcp"]
    }
  }
}
```

> ⚠ MCP 서버 (`lore mcp`) 는 v0.2 에서 출시됩니다. v0 사용자는 아래 "수동 컨텍스트" 절을 따르세요.

## 수동 컨텍스트 (v0)

`.lore/flows/*.md` 파일이 워크스페이스에 있으면 Claude Code 가 자동으로 인지합니다. 다음 한 줄을 `CLAUDE.md` 에 추가하면 강제 참조됩니다:

```md
도메인 맥락은 항상 `.lore/flows/*.md` 와 `.lore/DOMAIN_MAP.md` 를 참조해 답할 것.
```

## 권장 워크플로

1. 새 코드를 짤 때 `.lore/flows/<카테고리>.md` 를 사이드 패널에 띄워두기
2. 의미가 큰 변경 후 `lore sync` → 갱신된 마크다운이 자동 반영
3. 회사 위키와 연결하려면 `lore publish` → [Lore Board](/concepts/lore-board)

## 어노테이션 강제 (선택)

vibe coding 중 LLM 이 어노테이션을 빠뜨리는 것을 방지하려면 Claude Code 의 hook 시스템 (`PostToolUse`, `SessionStart`, `Stop`) 으로 자기 교정 루프를 만들 수 있습니다. **opt-in** — 팀 성향에 맞춰 채택. 셋업 가이드: [AI 코딩 하네스](/guides/ai-harness).
