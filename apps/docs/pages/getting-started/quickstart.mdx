# 5분 Quickstart

이 가이드는 `lore init` 부터 첫 `.lore/flows/<카테고리>.md` 가 생성되는 시점까지를 다룹니다. 평균 60~90초.

<Steps>

### CLI 설치

```bash
npm i -g lore-ai
```

### 프로젝트 초기화

```bash
cd my-project
lore init
```

생성되는 것:

- `lore.config.yaml` — 프로젝트·도메인 정의
- `.lore/flows/` — L2 카테고리 마크다운이 들어갈 자리
- `.lore/DOMAIN_MAP.md` — L1 도메인 지도

<Callout>
  서버와 앱이 **별도 저장소**로 분리되어 있다면, `lore init` 을 어느 쪽에 할지 / 두 번 할지가 셋업의 핵심입니다. → [Multi-repo 가이드](/guides/multi-repo)
</Callout>

### 도메인 부트스트랩

코드베이스에서 evidence 를 자동 수집하고, AI 로 도메인 맵 초안을 만듭니다.

```bash
lore bootstrap | pbcopy   # macOS — 프롬프트가 클립보드로
# 또는
lore bootstrap > prompt.md
```

생성된 프롬프트를 Claude Code / Cursor 에 붙여넣으면 두 파일의 diff 를 돌려줍니다:

- `lore.config.yaml` 의 `domains:` 블록 (영문 key + 한국어 label)
- `.lore/DOMAIN_MAP.md` (도메인 한 줄 설명 + subdomain 목록)

자세한 옵션 (`--heuristic-only`, `--out`) 과 동작 원리 → [CLI: bootstrap](/cli/bootstrap).

<Callout>
  AI 없이 빠른 초안만 원하면 `lore bootstrap --heuristic-only` 로 폴더 구조에서 도메인 후보를 정적 추출. 한국어 label 과 subdomain 정제는 사람이.
</Callout>

응답을 검토 후 `lore.config.yaml` 과 `.lore/DOMAIN_MAP.md` 에 반영하세요.

### 검증

```bash
lore check $(git ls-files '*.py' '*.ts' '*.tsx')
```

<Callout type="info">
  precommit 훅은 [hooks 가이드](/integrations/cursor#precommit) 참조.
</Callout>

### Sync

```bash
lore sync
```

`.lore/flows/auth.md`, `subscription.md`, `INDEX.md` 가 생성됩니다.

</Steps>
